# Summary

This video explores how Uber engineered a robust payment system capable of handling trillions of transactions without errors. The presentation describes Uber's evolution from a monolithic architecture to microservices, highlighting critical failures in their original payment infrastructure including instability, fragmentation, slow transactions, and reconciliation nightmares. To solve this, Uber rebuilt their payment system using a ledger-based architecture employing double-entry accounting principles where every transaction creates balanced credit and debit entries. The engineers created LedgerStore, an application layer above NoSQL databases that implements three core concepts: ceilings (immutable time ranges), manifests (security measures), and revisions (correction tracking). The video details their migration strategy for 250 billion historical records using checkpoint-based batch processing, then describes their Shadow Writer technique for gradually migrating to a new database using synchronous writes to the primary database and asynchronous writes to the secondary. The content also explains sophisticated indexing strategies—strongly consistent indexes for credit authorizations, eventually consistent indexes for non-critical data, and time-range indexes for archival management. This technical deep-dive targets software engineers and system designers interested in large-scale distributed systems and financial technology infrastructure.
