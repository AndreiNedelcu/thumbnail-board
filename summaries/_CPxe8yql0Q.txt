NASA's Voyager probes, launched in the 1970s, continue operating 15 billion miles away using original 50-year-old computers with only 70 kilobytes of memory—a remarkable feat of engineering efficiency. When Voyager 1 recently experienced degraded telemetry and aging thrusters, engineers employing expertise in antiquated programming languages like Fortran and assembly developed a software update to address these issues. The video explains Voyager's unique hardware architecture, including its plated wire memory system that physically stores binary data through magnetic fields, making it persistent even without power. Rather than running traditional operating systems, Voyager uses machine code, pseudo-code shortcuts, and an interrupt-driven system that allows NASA to remotely update its software despite 16-bit-per-second communication delays. Through the Deep Space Network, updates compile into machine code and flip individual memory bits to implement new instructions. The video illustrates how this aging system survives through periodic patches, including a critical 1995 update that later saved Voyager 1 when hardware malfunctioned in 2014. This content appeals to space enthusiasts, engineers, and those fascinated by how humanity maintains 1970s technology in the harsh environment of interstellar space.
