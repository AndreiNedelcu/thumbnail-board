This podcast episode examines the challenges developers face managing the flood of AI-generated code, particularly from tools like GitHub Copilot and Claude. The hosts argue that while AI code generation produces decent output, the review overhead often negates productivity gains. They critique the common practice of breaking code into tiny pull requests to make review manageable, noting this creates merge chaos and doesn't solve the fundamental problem. A key argument is that AI-generated code frequently requires rework, meaning developers end up doing work twice—once prompting and reviewing AI, then again fixing issues. The speakers reference Ghostly's open-source AI usage policy requiring human understanding and accountability, contrasting it with private companies that merely track token usage. They contend that senior developers use less AI not from resistance but pragmatic recognition that it slows them down, and suggest developers maintain the mental model of features before reviewing AI code. The episode targets mid-to-senior developers skeptical of over-relying on code generation tools, advocating for writing code themselves for better quality and faster overall delivery.
