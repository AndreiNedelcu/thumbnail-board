Discord faced a critical infrastructure challenge when its user base grew from 10 million to 140 million users between 2017 and 2021, requiring storage of trillions of messages with zero downtime. Initially using MongoDB, then Cassandra, Discord struggled with severe scaling issues—the distributed database's partitioning by channel and time bucket created hotspots that slowed read operations, especially during high-traffic events. The Java garbage collector added performance overhead, and maintenance tasks like compaction became prohibitively expensive. Discord solved this through a multi-pronged approach: migrating to ScyllaDB, a high-performance C++-based NoSQL database compatible with Cassandra but without garbage collection overhead; building data services in Rust using gRPC to reduce database load through request coalescing; and implementing intelligent routing by channel ID. A Rust-based migration tool accelerated the data transfer from an estimated three months to just nine days, moving 3.2 million messages per second. The results were dramatic—latency dropped from 40-125ms to 15ms for reads, and write latency stabilized at five milliseconds. This video targets software developers and tech enthusiasts interested in large-scale system design, database optimization, and how major platforms handle infrastructure challenges.
